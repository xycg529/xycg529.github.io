关于Hexo的一些事

发表于 2022-05-18 分类于 杂记

这两天被Hexo各种各样的问题折磨坏了。

起因是在原先Github Page的基础上加了个域名，然后莫名其妙网站的排版就乱了，找不到解决方法，遂推倒重来。

将之前的Fluid主题换成了现在的NexT主题，虽然没有之前美观，但访问速度是上去了。

然后又因为LateX公式的渲染的问题，以及莫名其妙的访问速度变慢，重开了不知道多少次，现在总结下出现的问题：

因为NexT自带支持了MathJax，所以无需安装插件包。

按照NexT官网的教程修改配置后，电脑上访问可以正常出现LateX公式，但手机访问并没有。

这个问题需要安装插件hexo-filter-mathjax，问题就在于安装插件上。

如果使用指令npm install hexo-filter-mathjax --save，可能会返回如下的提示：

尝试使用其建议的方法也都是类似的提示。

解决方法是更换淘宝的镜像，感觉是经典的网络问题需要国内镜像解决。=_=||

通过指令npm install -g cnpm --registry=https://registry.npm.taobao.org安装cnpm，以后用cnpm代替npm安装插件更好。

这样问题就算解决了。

之前试过了各种各样的插件，大概是hexo-renderer-pandoc导致hexo clean && hexo g时的速度变慢。

现在电脑和手机的访问速度都算正常，已经不敢随便安装插件了。

因为适配这个NexT主题我又改了文章的front-matter，导致时间错乱失去了原来的时间排序。=_=||

现在还有的问题是手机访问网页的LateX公式如果太长不换行会超出网页的限制，初步认为是LateX语法的问题。

希望之后的网站维护少些问题，已经感受到了小白做维护的痛苦。
