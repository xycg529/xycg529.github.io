发表于 2022-06-06 更新于 2022-06-07 分类于 数据结构

基本概念

查找——在数据集合中寻找满足某种条件的数据元素的过程称为查找。

查找表（查找结构）——用于查找的数据集合称为查找表，它由同一类型的数据元素（或记录）组成。

关键字——数据元素中唯一标识该元素的某个数据项的值，使用基于关键字的查找，查找结果应该是唯一的。

对查找表的常见操作

  1. 查找符合条件的数据元素——静态查找表，仅需关注查找速度。

  2. 插入删除某个数据元素——动态查找表，除了查找速度也要考虑增删操作是否方便实现。

查找算法的评价指标

查找长度——查找过程中需要对比关键字的次数。

平均查找长度——所有查找过程中进行关键字的比较次数的平均值。

顺序查找

从头到尾挨个查找，时间复杂度。

优化

对有序表进行顺序查找可以优化，比如升序表中当前关键字大于目标值则可以提前结束。

每个关键字被查概率不相等时，可以将关键字按概率从大到小排列。

折半查找（二分查找）

仅适用于有序的顺序表，时间复杂度。

分块查找

把顺序表连续的区间分成几块，每块中的元素都小于某个最大值，这些最大值可以构成一个索引表，索引表保存每个分块的最大关键字和分块的存储区间。

查找目标值时，只需要通过索引表确定指定的存储区间，在指定区间内进行顺序查找。

对于升序的索引表一般用二分查找来确定分块位置，若索引表中不包含目标关键字，则二分查找最终停在low>high，要在low所指分块中查找。因为最终low左边一定小于目标关键字，high右边一定大于目标关键字。而分块存储的索引表中保存的是各个分块的最大关键字。

树的定义

又称多路平衡查找树，树中所有结点的孩子个数的最大值称为树的阶，通常用表示。一棵阶树或为空树，或为满足如下特性的叉树：

  1. 树中各个结点至多有棵子树，即至多含有个关键字。

  2. 若根结点不是终端结点，则至少有两棵子树。

  3. 除根结点外的所有非叶结点至少有棵子树，即至少含有个关键字。

  4. 所有的叶结点都出现在同一层次上，并且不带信息（可以视为外部节点或类似于折半查找判定树的查找失败结点，实际上这些结点不存在，指向这些结点的指针为空）。

树的插入和删除

比如对阶树进行插入，每个结点的关键字个数，当时，（省略失败结点）。

插入新关键字后，若导致原结点关键字超过上限，则从中间位置将其中的关键字分为两部分，左部分包含的关键字方在原结点中，有部分包含的新关键字放到新结点中，中间位置的结点插入原结点的父结点。

若此时导致其父节点的关键字个数也超过了上限，则继续进行这种向上分裂的操作，直至这个过程传到根结点为止，进而导致树高度增加。

新元素一定是插入到最底层的“终端结点”，否则失败结点就不会在同一层，不满足树的特性。

若被删除关键字在终端结点，则直接删除该关键字（要注意结点关键字个数是否低于下限）。

若被删除关键字在非终端结点，则用直接前驱或直接后继来替代被删除的关键字。

若被删除关键字所在结点删除后的关键字个数低于下限，且与此结点右（左）兄弟结点关键字个数还很宽裕，则需要调整该结点、右（左）兄弟结点及其双亲结点（父子换位法）。

若兄弟结点也不够调整，则将关键字删除后与兄弟结点及其关键字合并。

在合并过程中，双亲结点中的关键字个数会减。若其双亲结点是根结点且关键字个数减少至（根结点关键字个数为时有棵子树），则直接将根结点删除，合并后的新结点成为根。

若双亲结点不是根结点，删完又有新的结点包含关键字树不满足要求，则需要一直调整或合并直到满足树性质为止。

对非终端结点的删除必能转换为对终端结点的删除。

树

树与分块查找类似，它满足下列条件：

  1. 每个分支阶段最多有棵子树
  2. 非叶根结点至少有两棵子树，其他每个分支结点至少有棵子树。
  3. 结点的子树个数与关键字个数相等。
  4. 所有叶结点包含全部关键字及指向相应记录的指针，叶结点中将关键字按大小顺序排列，并且相邻叶结点按大小顺序相互链接起来。
  5. 所有分支结点中仅包含它的各个子结点中的关键字的最大值及其指向子结点的指针。

树的查找可以通过根结点依次向下遍历查找（与二叉排序树类似），也可以直接顺序查找叶子结点。
